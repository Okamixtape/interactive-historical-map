# Workflow IA pour interactive-historical-map

## Configuration Git

### Branches
- **Locale (travail)** : `feature/limoges-map`
- **Distante (principale)** : `origin/master`

### Commandes standardisées

**Push vers master** :
```bash
git push origin feature/limoges-map:master
```

**Vérification avant push** :
```bash
git branch -vv
# Attendu : feature/limoges-map [origin/master]
```

**Synchronisation** :
```bash
git fetch origin master
git rebase origin/master
```

## Règles pour les assistants IA

### ❌ Interdictions
- Créer des branches `claude/*` ou `windsurf/*`
- Pousser sur des branches autres que `master`
- Modifier la configuration de tracking sans confirmation

### ✅ Bonnes pratiques
- Toujours vérifier `git status` avant de pousser
- Utiliser `git push origin HEAD:master` en cas de doute
- Demander confirmation si la branche tracking ne correspond pas

## Assistants utilisés

- **Claude Code** : Architecture, review, optimisations
